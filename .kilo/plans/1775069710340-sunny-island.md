# Plan: Taildrop Daily Database Backup (Pi → Unraid)

## Overview

Use Tailscale's built-in **Taildrop** file transfer to back up `prices.db` from the Pi to Unraid every day. No SSH keys, no NFS/SMB mounts, no extra software — just the Tailscale CLI both machines already have.

## How Taildrop works

- **Sender (Pi):** `tailscale file cp <file> <unraid-hostname>:` — pushes the file
- **Receiver (Unraid):** `tailscale file get --loop --wait <dir>` — runs continuously, auto-accepts incoming files into a directory

## Changes

### 1. Rewrite `farmprices/deploy/backup.sh` (Pi side)

The existing `backup.sh` does an unsafe `cp` of the live DB. Replace it with:

1. **Safe SQLite snapshot** using `sqlite3 ... ".backup ..."` — guarantees a consistent copy even if the app is writing at that moment
2. **Timestamp the filename** — `prices_2026-04-01_0200.db`
3. **Send via Taildrop** — `tailscale file cp <file> <unraid-hostname>:`
4. **Local rotation** — keep 7 days locally on the Pi (space is limited), the full 30-day archive lives on Unraid
5. **Log** results to `backups/backup.log`

```bash
#!/bin/bash
set -euo pipefail

# ── Config (edit these) ──────────────────────────────
DB_PATH="/home/richowen/Inventory-Prices/farmprices/prices.db"
BACKUP_DIR="/home/richowen/Inventory-Prices/farmprices/backups"
UNRAID_HOST="unraid"                    # ← Tailscale hostname of Unraid
LOCAL_KEEP_DAYS=7
LOG="$BACKUP_DIR/backup.log"
TS=$(date +%Y-%m-%d_%H%M)
BACKUP_FILE="$BACKUP_DIR/prices_${TS}.db"

mkdir -p "$BACKUP_DIR"
log() { echo "$(date '+%Y-%m-%d %H:%M:%S') $*" | tee -a "$LOG"; }

# 1. Safe SQLite backup
if [ ! -f "$DB_PATH" ]; then
    log "ERROR: DB not found at $DB_PATH"
    exit 1
fi
sqlite3 "$DB_PATH" ".backup '$BACKUP_FILE'"
log "Local backup: $BACKUP_FILE ($(du -h "$BACKUP_FILE" | cut -f1))"

# 2. Send to Unraid via Taildrop
if tailscale file cp "$BACKUP_FILE" "${UNRAID_HOST}:"; then
    log "Taildrop: sent to $UNRAID_HOST"
else
    log "ERROR: Taildrop send failed (is $UNRAID_HOST online?)"
fi

# 3. Rotate local backups (keep 7 days on Pi)
find "$BACKUP_DIR" -name "prices_*.db" -mtime +$LOCAL_KEEP_DAYS -delete
log "Local rotation: kept last $LOCAL_KEEP_DAYS days"
```

### 2. Cron job on the Pi

```bash
crontab -e
# Add:
0 2 * * * /home/richowen/Inventory-Prices/farmprices/deploy/backup.sh
```

Runs at 2 AM daily. The script is idempotent — safe to re-run manually.

### 3. Unraid: Taildrop receiver setup

Unraid needs a persistent process to accept incoming files. This depends on how Tailscale is installed on the Unraid box:

**Option A — Tailscale plugin (most common on Unraid):**

Create a User Script in Unraid (via the "User Scripts" plugin) set to run **At Array Start**:

```bash
#!/bin/bash
RECEIVE_DIR="/mnt/user/backups/farmprices"
mkdir -p "$RECEIVE_DIR"
# --loop: keep running, --wait: block until a file arrives
# --conflict=overwrite: each day's file has a unique timestamp so no conflicts
tailscale file get --loop --wait --conflict=overwrite "$RECEIVE_DIR" &
```

This starts a background process that auto-accepts all Taildrop files into `/mnt/user/backups/farmprices/`.

**Option B — Tailscale Docker container:**

If Tailscale runs as a Docker container, `tailscale file get` needs to run inside the container and the receive directory needs to be a mapped volume. The User Script would be:

```bash
#!/bin/bash
docker exec tailscale tailscale file get --loop --wait --conflict=overwrite /receive &
```

Where `/receive` is mapped to `/mnt/user/backups/farmprices` in the container's volume config.

### 4. Unraid: 30-day rotation

Add a second User Script on Unraid, scheduled **daily** (via Custom Cron: `0 3 * * *`):

```bash
#!/bin/bash
find /mnt/user/backups/farmprices -name "prices_*.db" -mtime +30 -delete
```

## Prerequisites (manual one-time setup)

1. **Pi:** Confirm `tailscale` CLI works: `tailscale status` — should show the Unraid node
2. **Pi:** `sqlite3` — the script will auto-install it via `apt-get` if missing
3. **Unraid:** Confirm Tailscale is installed and the node appears in `tailscale status`
4. **Unraid:** Enable Taildrop in Tailscale admin console (https://login.tailscale.com/admin/settings/features) if not already on
5. **Unraid:** Create the receive directory and start the `tailscale file get --loop` process
6. **Unraid:** Set up the 30-day rotation User Script
